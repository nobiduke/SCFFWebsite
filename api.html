Intro

Welcome to the scff python framework for the league of legends api.

User

The main class which is used as the client through which all other commands are used.

User.DataHold User.headers User.key
createSummoner() User.createDataHold() User.updateRank() User.getRankedSolo() User.getCurrentMatchBySummoner() User.getCurrentMatchByName() User.getMatchByNumber() User.getPlayerFromMatch() User.getPastGames() User.close()

Constructor

User(*RIOT_TOKEN_AS_STRING*)

The User constructor is required for all other commands since headers, data, and the key are required.

Requires: Token: str

Returns: User: User

If the key is invalid or the servers are down, then an error is thrown.

User.DataHold

The DataHold attribute can be used to implement the datadragon libraries with you code easily to name champions, spells, runes, and images.

Type: self.DataHold: DataHold

This attribute can be ignored if champion information is being ignored and nondependent stats are accessed.

User.headers

The default headers for requesting links from the API. Shouldn't be changed, but it can be if needed.

Type: self.headers: dict

User.key

The key that is used with every single API call. If you don't have a licensed product with RIOT, this must be refreshed every 24 hours.

Type: self.key: str

User.createSummoner()

User.createSummoner(name, region="na1", default=None, regionalLink=None)

Creates a Summoner object from a given summoner name and region.

Requires: name: str, region: str

Optional: default: any - What is returned if the summoner doesn't exist.

Optional: regionalLink: str - Datahold can be passed alternatively for this, but the regional link is required for match history.

Returns: Summoner: Summoner

User.updateRank()

User.updateRank(summoner)

Updates the Summoner.ranks property of the passed in Summoner object.

Requires: summoner: Summoner

User.getRankedSolo()

User.getRankedSolo(summoner, default=None)

Updates the rank for a Summoner and then returns the ranked information for Ranked-Solo/Duo in the form of a RankInfo object.

Requires: summoner: Summoner

Optional: default: any - What is returned if the Summoner has no current Ranked-Solo/Duo info

Returns: RankInfo: RankInfo

User.getCurrentMatchBySummoner()

User.getCurrentMatchBySummoner(summoner, default=None)

Returns the current match of the given Summoner in the form of a LiveMatch object.

Requires: summoner: Summoner

Optional: default: any - What is returned if the Summoner isn't currently in a match

Returns: LiveMatch: LiveMatch

User.getCurrentMatchByName()

User.getCurrentMatchByName(name, region="na1", default=None)

Returns the current match of a Summoner from a given Summoner name in the form of a LiveMatch object.

Requires: name: str, region: str

Optional: default: any - What is returned if the Summoner isn't currently in a match

Returns: LiveMatch: LiveMatch

User.getMatchByNumber()

User.getMatchByNumber(summoner, index, default=None, queueType=None)

Returns the Match of a Summoner by a given index, index=0 is the most recent Match. Match object.

Requires: summoner: Summoner, index: int

Optional: default: any - What is returned if the Summoner isn't currently in a match

Optional: queueType: str - Allows for a specific type of match to be searched for. Must follow the names defined by the League API

Returns: Match: Match

If the given index is greater than 99 an error is thrown since the API only returns 100 matches a time.

User.getPlayerFromMatch()

User.getPlayerFromMatch(match, summoner, default=None)

Returns the Player of a certain match by a given summoner name or previously established Summoner object in the form of a Player object.

Requires: match: Match, summoner: Summoner or string

Optional: default: any - What is returned if the Summoner isn't in that match

Returns: Player: Player

User.getPastGames()

User.getPastGames(summoner, count=20, default=None, time=false, queueType=None)

Returns a list of Player objects from a given amount of previous matches.

Requires: summoner: Summoner, count: int

Optional: default: any - What is returned if the Summoner isn't in that match

Optional: time: bool - If True is passed, then an additional return value of the duration: int of all matches combined will be returned.

Optional: queueType: str - Allows for a specific type of match to be searched for. Must follow the names defined by the League API

Returns: matches: list of Player:Player

The default value for count is 20 because a free API key doesn't allow for more than 20 requests in the same second, so be careful with how you use this command.

User.close()
